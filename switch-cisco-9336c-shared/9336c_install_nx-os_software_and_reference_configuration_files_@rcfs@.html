    Installare il software NX-OS e i file di configurazione di riferimento (RCF)

    05/23/2023 Collaboratori

    Installare il software NX-OS e RCF su uno switch cluster Cisco Nexus 9336C- FX2

    Il software Cisco NX-OS e il file di configurazione di riferimento (RCF) devono essere installati sugli switch cluster Cisco Nexus 9336C-FX2.

    Prima di iniziare

    Prima di installare il software NX-OS e RCF sullo switch del cluster, devono sussistere le seguenti condizioni:

    • Il cluster deve essere completamente funzionante (non devono essere presenti errori nei log o problemi simili).

    • Se si installa solo NX-OS e si mantiene la versione corrente di RCF, è necessario aver controllato o impostato la configurazione di boot desiderata in RCF per riflettere le immagini di boot desiderate.

    • Se è necessario modificare la configurazione di avvio per riflettere le immagini di avvio correnti, è necessario farlo prima di riapplicare RCF in modo che venga creata un’istanza della versione corretta in caso di riavvio futuro.

    • È necessario disporre di una connessione console allo switch, necessaria per l’installazione di RCF.

    • Per le versioni supportate di ONTAP, NX-OS e RCF, consultare la tabella di compatibilità degli switch nella pagina degli switch Ethernet Cisco. Vedere "Switch Ethernet Cisco" per ulteriori informazioni.

    • Esistono dipendenze di comando tra la sintassi del comando in RCF e quella presente nelle versioni di NX-OS.

    • Per una documentazione completa sulle procedure di upgrade e downgrade dello switch Cisco sugli switch Cisco Nexus serie 9000, è necessario consultare le guide all’aggiornamento e all’aggiornamento del software disponibili sul sito Web di Cisco. Vedere "Switch Cisco Nexus serie 9000" per ulteriori informazioni.

    • È necessario disporre dell’RCF corrente.

    Configurazione iniziale

    Gli esempi di questa procedura utilizzano due nodi. Questi nodi utilizzano due porte di interconnessione cluster da 100 GbE e3a ed e3b, come per il controller A400. Vedere "Hardware Universe" per verificare le porte cluster corrette sulle piattaforme.

    Gli output dei comandi possono variare a seconda delle diverse versioni di ONTAP.

    Gli esempi di questa procedura utilizzano la seguente nomenclatura di switch e nodi:

    • I nomi dei due switch Cisco sono cs1 e cs2.

    • I nomi dei nodi sono cluster1-01 e cluster1-02.

    • I nomi LIF del cluster sono cluster1-01_clus1 e cluster1-01_clus2 per cluster1-01 e cluster1-02_clus1 e cluster1-02_clus2 per cluster1-02.

    • Il prompt cluster1::*> indica il nome del cluster.

    La procedura richiede l’utilizzo di entrambi i comandi ONTAP e Cisco Nexus 9000 Series Switches; i comandi ONTAP vengono utilizzati se non diversamente indicato.

    1. se AutoSupport è attivato su questo cluster, eliminare la creazione automatica del caso richiamando un messaggio AutoSupport: system node autosupport invoke -node * -type all -message MAINT=x h

      Dove x è la durata della finestra di manutenzione in ore.

    2. impostare il livello di privilegio su Advanced (avanzato), immettendo y quando viene richiesto di continuare:
      set -privilege advanced

    Prima di iniziare
